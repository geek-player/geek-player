编写代码

  • 实体类：参考表结构与自己的需要

  • 接口类：dao层下的接口（抽象方法、默认方法和静态方法）

  • public int insert (User user);
public int update (User user);
public int delete (int id);
public User selectById(int id);
public List<User> select();
public User login(@Param('username') String username,@Param('password') String password)

  • 映射文件：

    1. 映射文件路径一定与接口类路径一致

    2. 映射文件名与类名一致

    3. 映射文件中的namespace，写成接口类的全路径包名.接口类名、

    4. 映射文件中的 id 就是接口类中的方法名（id不可能重复）

    5. 映射文件中的parameterType，写的内容与接口类中的形参一致

      若形参的类型是基础类型或String，且只有一个，则默认可不写，可以自动映射

      若形参的类型是非基础类型或String，且只有一个，则需要在映射文件中写类名

      若形参的类型是两个及以上基础类型或String，映射文件建议使用@Param

    6. 映射文件中的 resultType，写的是返回值类型（insert、delete、update无此属性）

      若是一个基础类型或String，则可以不写

      若是一个类或者集合，则直接写类名

      要求：列名与属性名、列名对应的类型与属性名对应的类型一致，若不满足如上要求使用resultMap
